# Security Policy

## Reporting Security Issues

Do not open public issues for security vulnerabilities or privacy problems.

Email the maintainer or use GitHub private vulnerability reporting if enabled.

## No PHI in Reports

Security reports must not include real patient data. Use synthetic examples and describe the class of issue.

## Scope

Security-sensitive areas include:

- Local database encryption.
- Key handling.
- Audit log integrity.
- Export and backup workflows.
- AI provider policy enforcement.
- PHI redaction and logging.
- Access control and session locking.

## Current Maturity

This repository is early-stage. It is not ready for production PHI.
